- Hash verification only on detail reads (performance)
- Change records use typed taxonomy (12 explicit events)

Patterns That Work:
- Documentation-first approach prevents scope drift
- Small, atomic commits make history readable
- Semantic Tailwind tokens enable easy theme changes
- Convex real-time queries eliminate polling

Patterns to Avoid:
- Don't add dependencies without checking TECH_STACK.md
- Don't skip change records (audit trail breaks)
- Don't mutate genome (creates new version instead)
